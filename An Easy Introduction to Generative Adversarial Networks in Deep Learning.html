An Easy Introduction to Generative Adversarial Networks in Deep Learning

Learn about Deep Learning’s most powerful networks!

George Seif
Sep 14 · 6 min read

Generative Adversarial Networks (GANs) are a type of neural network architecture which have the ability to generate new data all on their own. The study of these GANs is a piping hot topic in Deep Learning because of their power. In just a few short years, they’ve gone from generating fuzzy numbers to creating photo-realistic images of faces.

How GANs work

Generative Adversarial Networks (GANs) are classified within the group of generative models. That means that they’re able to produce, i.e generate completely new, “valid” data. By valid data, we mean that the network’s output should be something that we would deem acceptable for our target.

To illustrate, consider an example where we wish to generate some new images for training an image classification network. Of course, for such an application we want our training data to be as realistic as possible, perhaps quite similar in style to other image classification training data sets.

The figure below shows an example of a set of images that a GAN has generated. They look quite real! If we weren’t told that they were computer-generated, we might have actually believed that a human collected these!

To do this, GANs are built out of 2 separate opposing networks: The Generator and the Discriminator. The generator is trained to create realistic-looking images when given only a noisy image array as input. The Discriminator is trained to classify whether the image is real or not.

The real power in GANs comes from the opposing training style they follow. The Generator network’s weights are learned based on the loss of the Discriminator. Thus, the Generator is pushed to be trained in such a way that for the images it generates, it is very hard to tell if they are real or not. At the same time that these images are looking more and more real, the Discriminator is getting better and better at telling which images are real or not, no matter how similar they may look to the naked eye.

From a technical point of view, the loss of the Discriminator will be the error from classifying which images are fake and which are real; we’re measuring its ability to distinguish real and fake images. The Generator loss will be based on how good it is at “fooling” the Discriminator with fake images — i.e the classification error of the Discriminator on only the fake images, since the Generator wants that to be as high as possible.

Thus, GANs build a sort of feedback loop where the Generator is helping to train the Discriminator and the Discriminator is helping to train the Generator. They both get better together. The diagram below helps illustrate this.

Train a GAN to generate numbers in Pytorch

Now we’ll go through an example of how we can build and train our own GAN in Pytorch! The MNIST dataset contains 60,000 training images of black and white digits ranging from 1 to 9 where each image is of size 28x28. It’s perfect for our use case as it’s still very commonly used for Machine Learning PoCs and is a well-established set.

We’ll start with our imports. All we need is Pytorch stuff!

Next, we’ll prepare our DataLoader for the training data. Remember, we want to be generating random digits for MNIST i.e from 0 to 9. As such, we’ll need to setup labels for those 10 digits.

Now we can build our networks. We start with the Discriminator network below. Recall that the Discriminator network classifies whether or not an image is real — it is an image classification network. Thus, our input is an image with the standard MNIST size: 28x28 pixels. We flatten this image into a single vector of length 784. The output is a single value indicating whether or not the image is a real MNIST digit.

Now on to the Generator below. The generator network is responsible for creating the actual images — and it can do this from an input of pure noise! For this example, we’re going to have our generator start from a vector — again this is just pure random noise — of length 100. From this vector, our Generator will output a vector of length 784, which we can later reshape into our standard MNIST 28x28 pixels.

To set up our training we’ll need a few things:

  • Loss function
  • An optimizer for each network
  • Number of epochs
  • Number of batches

Pytorch also requires that, if we want our network to run on GPU, we must explicitly move the model to it. All of that code is shown below.

Now for our training loop. Training loops in Pytorch most commonly consist of an outer loop going through the epochs and an inner loop going through the batches of data. The key part with training GANs is that we’ll need to update both the Generator and Discriminator in one loop. Check out the code below for training the GAN and Pytorch. The steps are described in more detail just under the code.

(1) We start by preparing our *real* image data for the discriminator. The input is a batch of real MNIST images. The output is a vector of all ones, since a 1 indicates that the image is real.

(2) Next, we’ll prepare our input vector for the generator such that we can generate fake images. Recall that our generator network takes an input vector of length 100, so that’s what we create here. The images.size(0) is for the batch size

(3) From our random noise data vector created in step (2) we can generate our fake image data bypassing the vector to the generator. This will be used in combination with our real data from step 1 to train the discriminator. Notice also that this time our labels vector is all zeros, since 0 represents the class label for fake images.

(4) Given the fake and real images along with their labels, we can train our discriminator for classification. The total loss will be the loss of the fake images + the loss of the real images

(5) Now that our discriminator has been updated we can use it to make predictions. The loss on those predictions will be backpropagated through the generator, such that the generator’s weights are updated specifically *according to how well it is fooling the discriminator*.
(5a) generate some fake images to make predictions on
(5b) use the discriminator to make predictions on the batch of fake images and save the output.

(6) Using the predictions from the discriminator, we train our generator. Notice that we use the _real_labels_ of all 1s as the target, since the target for our generator is to create images that look real and are predicted as 1! Thus a loss of 0 for the generator would correspond the discriminator predict all 1s.
